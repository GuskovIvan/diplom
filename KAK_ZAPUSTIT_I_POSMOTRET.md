# Как запустить и посмотреть проект

Этот файл написан для новичка. Просто идите по шагам сверху вниз.

## 1. Что уже есть в проекте

Проект состоит из нескольких частей:

- `Frontend` - интерфейс Kanban-доски в браузере.
- `Backend` - сервер на Node.js, который обрабатывает REST API и WebSocket.
- `PostgreSQL` - настоящая база данных, где хранятся пользователи, проекты, колонки, задачи и события.
- `Prisma` - инструмент для работы с базой данных из кода.
- `Vitest` - автоматические тесты, которые проверяют расчет позиций, события, конфликты и WebSocket.

Главная идея диплома:

- REST API используется для надежных операций: создать задачу, изменить задачу, удалить задачу, войти в аккаунт.
- WebSocket используется для мгновенной синхронизации: если задача изменилась в одной вкладке, другая вкладка получает событие без обновления страницы.

## 2. Самый быстрый способ посмотреть результат

Если сервер уже запущен, откройте в браузере:

```text
http://localhost:4000
```

Данные для входа:

```text
Владелец:       demo@example.com / demo12345
Администратор: admin@example.com / admin12345
Исполнитель 1: worker1@example.com / worker12345
Исполнитель 2: worker2@example.com / worker12345
```

После входа вы увидите Kanban-доску с колонками и карточками.

В новой версии интерфейс специально собран под защиту дипломной работы:

- `Доска` - главный сценарий работы с задачами и drag-and-drop.
- `Roadmap` - дополнительный взгляд на те же задачи по этапам проекта.
- `Объявления` - адресные сообщения участникам проекта.
- `Аудит` - самое важное для защиты: серверные события, метрики синхронизации, задержка WebSocket и конфликты версий.

Если нужно показать преподавателю техническую ценность проекта, после перемещения карточки откройте вкладку `Аудит`: там видно, что действие сохранено на сервере, попало в журнал и доставляется через realtime-механизм.

Если владелец или администратор добавляет в команду email, которого еще нет в системе, аккаунт создается автоматически:

```text
Пароль можно задать в форме "Добавить пользователя". Если оставить поле пароля пустым, сервер создаст уникальный временный пароль и покажет его в уведомлении.
```

## 3. Если сайт не открывается

Откройте PowerShell в папке проекта:

```text
C:\PythonProjects\diplom
```

Запустите сервер:

```powershell
npm.cmd run start
```

После этого снова откройте:

```text
http://localhost:4000
```

Важно: в PowerShell лучше писать именно `npm.cmd`, а не просто `npm`, потому что на Windows иногда запрещен запуск `npm.ps1`.

## 4. Как запустить проект в режиме разработки

Режим разработки удобен, если вы меняете код и хотите видеть изменения быстрее.

Команда:

```powershell
npm.cmd run dev
```

После запуска будут доступны:

```text
Frontend: http://localhost:5173
Backend:  http://localhost:4000/api
```

Обычно для просмотра удобнее открывать:

```text
http://localhost:5173
```

## 5. Как проверить, что база данных работает

Откройте в браузере:

```text
http://localhost:4000/api/health
```

Если все хорошо, вы увидите примерно такой ответ:

```json
{
  "ok": true,
  "database": "postgresql"
}
```

Это значит, что сервер подключился к PostgreSQL.

## 6. Как проверить работу WebSocket

Это самый важный пункт для темы диплома.

1. Откройте `http://localhost:4000` в первой вкладке браузера.
2. Войдите как владелец `demo@example.com`.
3. Откройте `http://localhost:4000` во второй вкладке браузера.
4. Войдите как исполнитель `worker1@example.com`.
5. В первой вкладке назначьте исполнителю задачу или перетащите уже назначенную карточку.
6. Посмотрите на вторую вкладку.

Если все работает правильно, карточка во второй вкладке обновится без перезагрузки страницы.

Это и есть демонстрация синхронизации доски задач в реальном времени.

## 7. Что можно показать преподавателю

На защите или консультации можно показать такой сценарий:

1. Открыть приложение.
2. Войти под демо-пользователем.
3. Показать Kanban-доску.
4. Открыть вторую вкладку.
5. Перетащить карточку в первой вкладке.
6. Показать, что во второй вкладке изменение появилось автоматически.
7. Открыть вкладку `Аудит` и показать событие `TASK_MOVED`, задержку WebSocket и журнал действий.
8. Создать новую задачу через форму на доске.
9. Добавить участника или группу и показать, что это тоже попадает в журнал событий.
10. Открыть `http://localhost:4000/api/health` и показать, что используется PostgreSQL.
11. Запустить `npm.cmd test` и показать, что автоматические тесты проходят.
12. Открыть файл `prisma/schema.prisma` и показать структуру базы данных.

## 8. Как посмотреть REST API

Некоторые адреса можно открыть прямо в браузере.

Проверка сервера и базы:

```text
http://localhost:4000/api/health
```

Остальные запросы требуют авторизации, поэтому их удобнее смотреть через код приложения или через инструменты вроде Postman/Insomnia.

Основные API описаны здесь:

```text
docs/api.md
```

## 9. Как посмотреть структуру базы данных

Главный файл модели базы:

```text
prisma/schema.prisma
```

Там описаны таблицы:

- `User` - пользователи.
- `Project` - проекты.
- `ProjectMember` - участники проектов.
- `BoardColumn` - колонки Kanban-доски.
- `TaskGroup` - группы исполнителей.
- `TaskGroupMember` - участники групп исполнителей.
- `Task` - карточки задач.
- `TaskEvent` - журнал событий для синхронизации.

Миграция базы лежит здесь:

```text
prisma/migrations/20260417120000_init/migration.sql
```

## 10. Как открыть базу данных визуально

Самый простой вариант - установить pgAdmin или DBeaver.

Параметры подключения:

```text
Host:     localhost
Port:     5432
Database: kanban_diplom
User:     kanban
Password: kanban_password
```

После подключения можно смотреть таблицы:

- `User`
- `Project`
- `BoardColumn`
- `TaskGroup`
- `TaskGroupMember`
- `Task`
- `TaskEvent`

Особенно полезно открыть таблицу `TaskEvent`: там видно, какие события создаются при изменении доски.

## 11. Как заново подготовить базу

Обычно это не нужно, но если база пустая или что-то сломалось, выполните:

```powershell
npm.cmd run prisma:deploy
npm.cmd run db:seed
```

Первая команда применяет структуру базы данных.

Вторая команда создает демо-пользователя и тестовую доску.

## 12. Как собрать проект

Сборка нужна, чтобы проверить, что проект компилируется без ошибок.

Команда:

```powershell
npm.cmd run build
```

Если ошибок нет, значит TypeScript-код и frontend успешно собираются.

## 13. Как запустить тесты

Команда:

```powershell
npm.cmd test
```

Сейчас тестируются:

- алгоритм расчета позиции карточки при перемещении по доске;
- события участников и групп;
- увеличение версии задачи при перемещении;
- обнаружение конфликта версий;
- перемещение внутри одной колонки;
- доставка события через Socket.IO.

Это относится к теме конфликтов и синхронизации Kanban-доски.

## 15. Что делать, если порт занят

Если при запуске появляется ошибка, что порт `4000` уже используется, скорее всего сервер уже запущен.

Проверьте в браузере:

```text
http://localhost:4000
```

Если сайт открывается, ничего делать не нужно.

Если нужно остановить старый сервер, закройте окно PowerShell, где он был запущен.

## 16. Что говорить простыми словами

Можно объяснять проект так:

```text
Я разработал веб-приложение для управления проектными задачами по Kanban-модели.
REST API используется для надежного изменения данных и сохранения их в PostgreSQL.
WebSocket используется для доставки событий другим подключенным пользователям.
Когда один пользователь перемещает карточку, сервер сохраняет изменение в базе,
создает событие и отправляет его всем участникам этой доски.
```

Коротко:

```text
REST отвечает за надежность, WebSocket отвечает за скорость синхронизации.
```
